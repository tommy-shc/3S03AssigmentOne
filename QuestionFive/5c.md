Through this exercise, it has become clear that to achieve high test coverage there is a efficient method by combing the strength of AI's wide coverage in conjunction with human input to refine the tests. Gen AI excelled at providing code to meet the exhaustive scenarios for certain key parts such as testing all parts of a switch statement, or different permutations of input data that would take a human considerably more time, however, it is also clear that it doesn't replace human work in the testing process. The human cognition remains essential for the integration of the tests into the project as well as deep logical cases that the AI will never have the full context of. As an example, during the testing process, the AI failed to generate tests for the scenario in which the user adds an item to the cart, after paying. The AI failed to follow the flow of the project and identify that particular vulnerability in the testing suite. This deep context that the human cognition has makes the test suite meaningful to the project, and helps improve the overall maintainability of the project.

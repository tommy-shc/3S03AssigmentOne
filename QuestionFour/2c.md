The functionality of calculator is not complete. Since the test suite started with only two test cases, we developed a calculator that could check for two initial cases, divide by zero, and large number overflow. Our divide by zero test checked that it would correctly resolve a 0 to NaN, but fails to account for special cases of infinity but some standards. For example, 0/0 should be NaN but special cases such as -100/(Really small fraction) can be considered negative infinity, while 100/(Really small fraction) is considered positive infinity, two completely different solutions. The overflow test checks for overflow, but we lack an underflow test to check for the flipside of calculations. Additionally, the calculator lacks feedback. Although, we can correctly handle most error cases, it fails to prompt the user about what went wrong, such as if the numerator is NaN, or if the denominator is NaN, etc. In a traditional development project, these special cases would be accounted for at the start, resulting in a much bulkier calculator class.

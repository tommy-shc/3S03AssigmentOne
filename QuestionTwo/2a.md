### Program 1:

The issue with program one is an off by one error. Instead from the end of the array x to the beginning, it goes from the end to the first element since the condition in the for loop has a > 0 instead of a >= 0. here is the corrected code:

``` java
/**
* Find last index of element
*
* @param x array to search
* @param y value to look for
* @return last index of y in x; -1 if absent
* @throws NullPointerException if x is null
*/
public static int findLast(int[] x, int y) {
    for (int i=x.length-1; i >= 0; i--) {
        if (x[i] == y) {
            return i;
        }
    }
    return -1;
}
```

### Program 2:

The problem with program two is the program will return it's value as soon as it sees the first zero, instead of waiting until it's sure the zero it saw was the last zero in the array. An easy way of fixing this would be to store the index of the last seen zero in a variable:

``` java
/**
* Finds last index of zero
*
* @param x array to search
*
* @return last index of 0 in x; -1 if there is no zero
* @throws NullPointerException if x is null
*/
public static int lastZero(int[] x) {
    int last_seen = -1;
    
    for (int i = 0; i < x.length; i++) {
        if (x[i] == 0) {
            last_seen = i;
        }
    }
    return last_seen;
}
```

### Program 3:

The problem with program three is that it also includes the number 0 as being positive. We can easily correct this by changing the comparison from >= 0 to just > 0:

``` java
/**
* Counts positive elements in array
*
* @param x array to search
* @return number of positive elements in x
* @throws NullPointerException if x is null
*/
public static int countPositive(int[] x) {
    int count = 0;
    for (int i=0; i < x.length; i++) {
        if (x[i] >= 0) {
            count++;
        }
    }
    return count;
}
```

### Program 4:

The problem with program four is that it doesn't account for the number being a negative odd number, only a positive odd number. We can fix this by changing the condition to check if x[i]%2 == -1 instead of 1. We do not need to check if it is also 1 since the second condition checks for all positive numbers:

``` java
/**
* Count odd or positive elements in an array
*
* @param x array to search
* @return count of odd or positive elements in x
* @throws NullPointerException if x is null
*/
public static int oddOrPos (int[] x) {
    // Effects: if x is null throw NullPointerException
    // else return the number of elements in x that
    // are either odd or positive (or both)
    int count = 0;
    for (int i = 0; i < x.length; i++) {
        if (x[i]%2 == -1 || x[i] > 0) {
            count++;
        }
    }
    return count;
}
```
